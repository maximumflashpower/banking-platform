# Runbooks

This directory is reserved for operational runbooks.

Examples of intended runbooks:

- ACH rail failure
- cards processor outage
- ledger mismatch investigation
- immutable audit chain verification
- backup and restore verification
- incident containment / breach simulation response

A runbook should include:

- trigger / when to use it
- prerequisites
- commands
- expected outcomes
- escalation conditions
- evidence to retain
